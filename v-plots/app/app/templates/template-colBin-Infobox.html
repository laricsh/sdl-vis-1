Each column a bin option for CSV upload

Requirements for Dataset File:

  • Comma separated CSV file.
  • Any kind of data is allowed for the binning as the histograms will be created based on height of the bins.

Creation of Histogram:

  • After CSV upload, the user needs to select a column from the drop-down menu as a filter. The drop-down menu is generated from the uploaded file.
  • The distinct values from the selected column is used to generate filter values for top/left and bottom/right distribution.
  • These filter values are used to group each column into two groups.
  • If the selected filter value does not exist in rest of the columns(other than filter column), those values from a column are selected that are not in the filter column.
  • The height of each column is calculated as: Number of rows in a column that has the selected filter value in the filter column / Total number of selected filter value in filter column

Example Dataset:

Group one two three
YES 5 NO 3
YES NO NO NO
NO NO NO 1.5
YES 8 7 NO
NO 4 7 NO

For e.g., for the given data set example, consider the user selects Group as a column to act as a filter. Here Group has YES and NO as distinct value. There can be more than 2 distinct values. Consider for top/left distribution, the user selects YES as a filter. Now, in the rest of the columns, there is no cell with YES value. In this case, for binning all columns based on YES, all those values will be selected which are not in the Group column. So for column one, 5 and 8 will be selected for filter value YES. Here the height of bin for column one would be: 2/3. Similarly, the height for each column will be calculated for this particular filter value.
Same process will follow for each column for a filter value selected for bottom/right distribution.
